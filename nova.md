## 1. NOVA 平台簡介

NOVA 是一套以大型語言模型（LLM）為核心的智能應用平台，結合多模型選擇、角色模擬、流程自動化與圖表視覺化功能，協助使用者快速建立 AI 助理流程、分析資料並完成特定任務。平台特點包括模組化設計、直覺操作介面，以及具備擴充性的流程設定機制。

---

## 2. 開始使用NOVA

### 2.1 帳號註冊與登入

1. 前往 NOVA 平台首頁，點選「註冊」或「登入」。
2. 輸入電子郵件與密碼建立帳號或以Google帳號登入。
3. 成功登入後即可進入主畫面，開始建立個人 [Workspace](#31-Workspace)。

### 2.2 Assistant 快速建立流程

您可以透過 Assistant 快速建立流程。透過直接與 Assistant 對話，描述您想要建立的工作內容。系統將依據您的敘述，自動產生一條初步的流程架構，供您後續調整與優化。

目前支援的預設範例包括：

* Project Manager
* Traveling Plan
* Team Retro
* Gantt Chart
* Flow Chart
* Standup Meeting

點擊任一範例，即可立即體驗！

### 2.3 權限功能
NOVA 目前提供基礎權限控制機制，協助使用者簡單管理協作範圍：
* 使用者登入後，可檢視並管理自己所建立的所有 Workspace。
* 任何持有特定 Workspace 連結的使用者，皆可進入該 Workspace 並參與協作作業。

### 2.4 多人協作
NOVA 支援多人即時協作功能，提升團隊協同效率與互動體驗：
* 多位使用者可同時在線上編輯同一個 Workspace，適合進行分工合作。
* 系統會於每位使用者的滑鼠游標上顯示其帳號名稱，方便識別參與者身份。
* 使用者亦可透過按下斜線「 / 」鍵，輸入文字與其他線上成員即時對話，促進溝通與協作。

## 3. 名詞與功能說明

### 3.1 Workspace
Workspace 為 NOVA 的核心單位，您可以視==專案、部門或任務==建立不同的 Workspace，以便清楚區分各項流程與資料。

在 Workspace 左側目錄欄中，點選 … 圖示後可選擇以下動作：
* Rename：變更 Workspace 名稱，方便辨識用途。
* Duplicate：複製現有 Workspace，快速建立相似架構的新專案。
* Export：匯出 Workspace 資訊，便於備份或分享。
* Delete：刪除該 Workspace，操作前請確認不再需要其內容。

### 3.2 Note
Note可作為每個 Chat 流程的輔助筆記區，可作為獨立筆記，便於協作與查閱。也可用來作為任務描述或指令輸入，將帶有指令的Note[串接](#35-Connect)至 [Chat](#33-Chat) ， AI 即依據指令執行對應任務。


### 3.3 Chat
Chat 是 NOVA 中==與 AI 互動==的基本單位，用來串接其他元件，建立任務對話流程：

* 您可逐一建立 Chat 視窗，與指定的語言模型（如 GPT-3、GPT-4）進行互動，輸入指令或任務內容並接收回應。
* Chat 可獨立運作，也可與其他模組[串接](#35-Connect)成完整流程。
* 若 Chat 與 [Role](#34-Role)[串接](#35-Connect)，則代表此任務由特定角色身份執行若與[Note](#32-Note) 相關聯，則表示任務說明由Note 發出並交由此 Chat 處理。
* 每個 Chat 視窗可視為流程中的一個步驟，可作為多步驟流程的一部分，與其他 Chat [串接](#35-Connect)運行。
* NOVA 提供多種類型的 Chat，依任務屬性進行功能強化與介面設計。Chat預設為Mino Chat，點擊Chat左下方圖示可切換為Mino Chat，使用時可根據實際情境選擇最適合的類別：
    * Text Chat
標準文字對話介面，適用於純文字溝通與一般表格顯示等應用情境。
     * Mino Chat
Mino Chat 是專門用來產出 [Mino](#36-Mino) 元件的 Chat 類型，其設計目的類似 No Code 工具，協助使用者快速建立可互動的元件。Mino Chat 通常不與其他 Chat 串接，而是作為元件生成器獨立運作。透過 Mino Chat 建立的元件可直接拖曳到畫面中使用，也可以透過「＋」按鈕加入 [Mino元件庫](#37-Mino元件庫)，供後續流程使用。

* Chat右上方的功能：
    * Share：點擊後可對當前 Chat 進行快照並產生固定連結，保存該時點的完整狀態（包含 Role 設定等），後續修改不會影響該連結。
    * Clear：點擊後可清除Chat中的所有對話紀錄。
    * Clear all chat history：Chat右上方的Clear，點擊後可清除。
    * Delete：刪除整個Chat視窗。
### 3.4 Role
您可自訂角色來模擬不同專業身份，例如「法務顧問」、「資深 HR」、「數據分析師」等。設定後，AI 將依據角色語氣與任務目標回應使用者，提供更具針對性的互動內容。

除了基本身份，您也可以為角色設定具體背景，以幫助 AI 更準確地理解角色立場與回答風格。例如：

> 「你是某科技新創公司的資深招募專員，擁有三年面談與履歷篩選經驗」
> 「你是某零售企業的數據分析師，擅長從營運數據中發掘問題，並以 4P 分析、AARRR 模型等架構協助團隊釐清產品策略與市場定位。」
> 「你是專注於科技業的自由職業 Headhunter，熟悉軟體工程師的招募與職涯諮詢」



### 3.5 Connect
NOVA 支援不同元件(如: [Chat](#33-Chat)、[Role](#34-Role)、[Note](#32-Note)、[Ｍino](#36-Mino))之間透過==「拉線連結」==進行上下文連結，將多個元件串接為具邏輯性的工作流程。例如串聯 Chat 元件，Chat A 的輸出內容可 ==自動==作為 Chat B 的輸入來源，形成跨角色、跨任務的資訊流動機制，適用於模擬多角色協作場景。拉線機制不僅有助於視覺化流程，也提升資訊在多步驟任務中的傳遞效率。

NOVA Connect的方式有以下三種：
* 方式一：節點拖曳連接
元件四周皆設有節點，使用者可點選任一節點後按住滑鼠左鍵，拖曳至另一個已存在的元件節點以建立連線。第一個點選節點為起點，放開滑鼠的位置為終點，連線完成後會顯示箭頭，表示資訊流向。
> 「從 Note 拖曳節點至 Chat，表示 Note 為起點、Chat 為終點。此連線代表 Note 的內容將傳遞至 Chat，常見應用如將任務描述寫入 Note，並交由 AI 在 Chat 中執行。」

* 方式二：點選節點自動連接並創建新元件
若目標元件尚未存在，使用者可點擊元件節點後選擇要產生的新元件類型。系統將自動建立新的元件並完成連線，簡化流程建構。


* 方式三：使用 @ 引用標記
在元件右上角可點選 @ 圖示，設定此元件需引用的來源元件，可同時引用多個來源。系統將自動建立從來源元件指向目前元件的連線，表示資料將由來源流向此處。元件本身為終點，引用的來源元件為起點。
> 「從 Chat 點選@，選擇引用 Role，表示 Chat 為終點，Role 為起點。此連線代表 Note 的內容將傳遞至 Chat，常見應用如將任務描述寫入 Note，並交由 AI 在 Chat 中執行。」

### 3.6 Mino
Mino 是 NOVA 平台中的可插入式功能元件，如檔案上傳器、圖表展示區、時鐘、輸入框等，支援資料輸入與觸發角色流程。使用者可透過Mino Chat產出Mino元件，Mino元件可直接拖曳到畫面中使用，也可以透過「＋」按鈕加入 [Mino元件庫](#37-Mino元件庫)，供後續流程使用。

Mino元件在畫面中作為獨立元件，也可透過Connect連接，藉由Chat輸入需要的資料，呈現在Mino上。

透過點擊Mino元件右上方之連結分享圖示，也可將Mino元件以獨立網址的方式顯示在瀏覽器上。

### 3.7 Mino元件庫
Mino 元件庫收錄了所有可用的互動元件，使用者可從中選擇、拖拉進 Workspace 畫面，自由配置任務所需工具。

### 3.8 Webhook
Nova 提供 Webhook 功能用於 Chat 或 Workflow 的資料串接，支持雙向資料流：
* Webhook-inward：提供 API Key 與 Webhook URL，允許外部系統推送資料至 Nova。
* Webhook-outward：配置外部系統的 Webhook URL 與 API Key，允許 Nova 推送資料至外部系統。

在畫面中創建Webhook-inward/Webhook-outward並與Chat/Workflow 拉線連結，即可使用。

### 3.9 API Pair
Nova 提供 API Pair 功能，用於配對 API Input 與 API Output，實現完整的 API 工作流程。
* API Input：接收外部系統透過 API 發送的資料，觸發連接的 Workflow。提供 API Key 與 Base URL 供外部系統調用。

* API Output：與 API Input 配對，用於接收 Workflow 的處理結果。作為工作流識別節點，指示資料應輸出至何處。

點擊API Pair圖示，在畫面中創建API Input和API Output視窗，並與Chat/Workflow 拉線連結，即可使用。

## 4. 使用情境示範

### 4.1 圖表與資料視覺化流程

情境：用戶欲查詢銷售資料並生成圖表

1. 串接工廠資料庫，取得即時生產數據 

2. AI自動進行資料視覺化： 

    * 前 20 筆關鍵紀錄自動呈現為長條圖 
    * 根據狀態、類別與數量生成堆疊圖 
    * 產出彩色表格，自動標記重要指標 
    * 製作趨勢圖，掌握設備或狀態變化 

   User可直接以自然語言查詢數據內容，AI會辨別自然語言，生成對應查詢語法，例如：「顯示本週需完成的緊急任務」 

3. AI 根據數據結果，自動生成維護建議，協助預測風險並提前處理 

### 4.2 履歷審核流程模擬

情境：協助 HR 團隊進行履歷初步篩選

1. 建立職缺描述（JD） 
2. 建立 Mino，將收到的履歷上傳至系統 
4. 由 AI 自動判讀履歷與職缺的匹配程度 
5. AI 自動產出面試邀請或婉拒信件 
6. HR 一鍵寄出，完成通知

## 5. 常見問題 FAQ

**Q：上傳資料後沒有反應？**  
請檢查是否重複上傳名稱相同的檔案，若重複，請更換檔名。

**Q：AI 無法回答特定問題？**  
可能為上下文不完整、角色描述不清或資料尚未串接。請嘗試補充筆記內容或重設對話上下文。
