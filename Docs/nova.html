Welcome to NOVA

This manual is designed for users who are new to the NOVA platform. Through this document, you will understand NOVA's basic concepts, operation methods, key terminology explanations, and common practical application scenarios.

1. NOVA Platform Overview

NOVA is an intelligent application platform with Large Language Models (LLM) as its core, combining multi-model selection, role simulation, workflow automation, and chart visualization features to help users quickly build AI assistant workflows, analyze data, and complete specific tasks. The platform's features include modular design, intuitive user interface, and an extensible workflow configuration mechanism.

2. Getting Started with NOVA

2.1 Account Registration and Login

  1. Go to the NOVA platform homepage and click "Sign Up" or "Login".
  2. Enter your email and password to create an account or log in using your Google account.
  3. After successful login, you will enter the main screen and can begin creating your personal Workspace.

2.2 Assistant Quick Workflow Creation

You can quickly create workflows through the Assistant. By directly communicating with the Assistant and describing the work content you want to create, the system will automatically generate an initial workflow structure based on your description, which you can subsequently adjust and optimize.

Currently supported default examples include:

  • Project Manager
  • Traveling Plan
  • Team Retro
  • Gantt Chart
  • Flow Chart
  • Standup Meeting

Click on any example to experience it immediately!

2.3 Permission Features

NOVA currently provides a basic permission control mechanism to help users easily manage collaboration scope:

  • After users log in, they can view and manage all Workspaces they have created.
  • Any user with a specific Workspace link can enter that Workspace and participate in collaborative work.

2.4 Multi-User Collaboration

NOVA supports multi-user real-time collaboration features to enhance team collaboration efficiency and interaction experience:

  • Multiple users can simultaneously edit the same Workspace online, which is suitable for collaborative work distribution.
  • The system displays each user's account name on their mouse cursor for easy identification of participant identities.
  • Users can also press the forward slash "/" key to input text and have real-time conversations with other online members, promoting communication and collaboration.

3. Terminology and Feature Explanations

3.1 Workspace

Workspace is the core unit of NOVA. You can create different Workspaces based on projects, departments, or tasks to clearly distinguish various workflows and data.

In the left sidebar directory of Workspace, click the "..." icon to select from the following actions:

  • Rename: Change the Workspace name for easy identification of its purpose.
  • Duplicate: Copy an existing Workspace to quickly create a new project with a similar structure.
  • Export: Export Workspace information for easy backup or sharing.
  • Delete: Delete the Workspace. Please confirm that you no longer need its contents before performing this action.

3.2 Note

Note can serve as an auxiliary note area for each Chat workflow and can be used as an independent note for collaboration and reference. It can also be used as a task description or instruction input. When a Note with instructions is connected to a Chat, the AI will execute the corresponding task according to the instructions.

3.3 Chat

Chat is the basic unit for interacting with AI in NOVA, used to connect other components and build task conversation workflows:

  • You can create Chat windows one by one and interact with specified language models (such as GPT-3, GPT-4), entering commands or task content and receiving responses.
  • Chat can operate independently or be connected with other modules to form a complete workflow.
  • If Chat is connected with a Role, it means this task is executed by a specific role identity. If associated with a Note, it means the task description is issued by the Note and handled by this Chat.
  • Each Chat window can be viewed as a step in a workflow and can be part of a multi-step workflow, connected with other Chats to run.
  • NOVA provides multiple types of Chat, with functionality enhancements and interface design based on task attributes. Chat defaults to Mino Chat. You can click the icon in the lower left of Chat to switch between types. When using, you can choose the most suitable category based on the actual situation:

Text Chat: Standard text conversation interface suitable for pure text communication and general table display applications.

Mino Chat: Mino Chat is a specialized Chat type designed to produce Mino components. Its design purpose is similar to No Code tools, helping users quickly create interactive components. Mino Chat is typically not connected with other Chats but operates independently as a component generator. Components created through Mino Chat can be directly dragged into the screen for use or added to the Mino Component Library through the "+" button for subsequent workflow use.

Functions in the upper right of Chat:

  • Share: Click to take a snapshot of the current Chat and generate a fixed link that preserves the complete state at that point (including Role settings, etc.). Subsequent modifications will not affect this link.
  • Clear: Click to clear all conversation records in Chat.
  • Clear all chat history: The Clear button in the upper right of Chat. Click to clear.
  • Delete: Delete the entire Chat window.

3.4 Role

You can customize roles to simulate different professional identities, such as "Legal Advisor", "Senior HR", "Data Analyst", etc. After setting, the AI will respond to users based on the role's tone and task objectives, providing more targeted interaction content.

In addition to basic identity, you can also set specific backgrounds for roles to help AI more accurately understand the role's perspective and response style. For example:

"You are a senior recruitment specialist at a certain technology startup with three years of experience in interviews and resume screening"
"You are a data analyst at a certain retail enterprise, skilled at discovering problems from operational data, and using frameworks such as 4P analysis and AARRR models to help teams clarify product strategy and market positioning."
"You are a freelance Headhunter focusing on the technology industry, familiar with software engineer recruitment and career consulting"

3.5 Connect

NOVA supports line-based connection between different components (such as Chat, Role, Note, Mino) to establish context linkage, connecting multiple components into a logically structured workflow. For example, when connecting Chat components, the output of Chat A can automatically serve as the input source for Chat B, forming an information flow mechanism across roles and tasks suitable for simulating multi-role collaboration scenarios. The line mechanism not only helps visualize workflows but also improves the efficiency of information transfer in multi-step tasks.

NOVA Connect has the following three methods:

Method 1: Node Drag Connection

Nodes are set around all components. Users can click any node, hold the left mouse button, and drag it to another existing component's node to create a connection. The first selected node is the starting point, and the position where the mouse is released is the endpoint. After the connection is complete, an arrow is displayed indicating the information flow direction.

"Dragging a node from Note to Chat means Note is the starting point and Chat is the endpoint. This connection means the content of Note will be passed to Chat, a common application is writing task descriptions in Note and having AI execute them in Chat."

Method 2: Click Node to Auto Connect and Create New Component

If the target component does not yet exist, users can click a component node and select the type of new component to generate. The system will automatically create the new component and complete the connection, simplifying workflow construction.

Method 3: Use @ Reference Mark

You can click the @ icon in the upper right of a component to set the source components that this component needs to reference. Multiple sources can be referenced simultaneously. The system will automatically create connections from the source components pointing to the current component, indicating that data will flow from the source to this location. The component itself is the endpoint, and the referenced source components are the starting points.

"Clicking @ from Chat and selecting to reference Role means Chat is the endpoint and Role is the starting point. This connection means the content of Note will be passed to Chat, a common application is writing task descriptions in Note and having AI execute them in Chat."

3.6 Mino

Mino is a pluggable functional component in the NOVA platform, such as file uploaders, chart display areas, clocks, input boxes, etc., supporting data input and triggering role workflows. Users can generate Mino components through Mino Chat. Mino components can be directly dragged into the screen for use or added to the Mino Component Library through the "+" button for subsequent workflow use.

Mino components serve as independent components on the screen and can also be connected through Connect, using Chat to input the needed data and display it on Mino.

By clicking the link sharing icon in the upper right of a Mino component, you can also display the Mino component as an independent URL in the browser.

3.7 Mino Component Library

The Mino Component Library contains all available interactive components. Users can select from it and drag them into the Workspace screen to freely configure the tools needed for tasks.

3.8 Webhook

Nova provides Webhook functionality for data integration in Chat or Workflow, supporting bidirectional data flow:

  • Webhook-inward: Provides API Key and Webhook URL, allowing external systems to push data to Nova.
  • Webhook-outward: Configure external system's Webhook URL and API Key, allowing Nova to push data to external systems.

Create Webhook-inward/Webhook-outward on the screen and pull line connections with Chat/Workflow to use them.

3.9 API Pair

Nova provides API Pair functionality to pair API Input and API Output, implementing a complete API workflow.

  • API Input: Receives data sent by external systems through API, triggering connected Workflow. Provides API Key and Base URL for external systems to call.
  • API Output: Pairs with API Input to receive processing results from Workflow. Serves as a workflow identification node, indicating where data should be output.

Click the API Pair icon to create API Input and API Output windows on the screen and pull line connections with Chat/Workflow to use them.

4. Usage Scenario Demonstrations

4.1 Chart and Data Visualization Workflow

Scenario: User wants to query sales data and generate charts

  1. Connect to factory database to retrieve real-time production data
  2. AI automatically performs data visualization:
    • First 20 key records are automatically presented as bar charts

Users can directly query data content using natural language. AI will identify the natural language and generate corresponding query syntax. For example: "Display urgent tasks that need to be completed this week"

  1. Based on data results, AI automatically generates maintenance recommendations to help predict risks and handle them in advance

Scenario: Help HR team conduct initial resume screening

  1. Create job description (JD)
  2. Build Mino to upload received resumes to the system
  3. AI automatically judges the matching degree between resume and job opening
  4. AI automatically generates interview invitations or rejection letters
  5. HR sends them out with one click to complete notifications

5. Frequently Asked Questions FAQ

Q: No response after uploading data?

Please check if you have uploaded files with the same name repeatedly. If so, please rename the file.

Q: AI cannot answer specific questions?

It may be due to incomplete context, unclear role description, or data not yet connected. Please try supplementing note content or resetting the conversation context.
